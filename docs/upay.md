# UPay

This document describes the **UPay REST API layer** added to the gateway.  
It exposes endpoints that wrap the existing UnionBank UPay integration services.

## Overview

The gateway now provides a dedicated **UPay module/controller**:

- **Module**: `src/modules/upay/upay.module.ts`
- **Controller**: `src/modules/upay/upay.controller.ts`
- **Service**: `src/modules/upay/upay.service.ts`
- **DTOs (validation + Swagger)**: `src/modules/upay/dto/upay.dto.ts`

These endpoints call the UnionBank integration service:

- `src/integrations/unionbank/services/unionbank-upay.service.ts`

## Base URL

```text
http://localhost:3000/api/v1
```

## Authentication & Headers

All endpoints in this module are protected (same as other domain modules):

- `Authorization: Bearer <access-token>` (required)
- `Content-Type: application/json` (for `POST`)
- `x-idempotency-key: <key>` (recommended for `POST`)
- `x-request-id: <id>` (optional; set automatically by middleware if missing)

## Endpoints

### Create UPay Transaction

Creates a UPay payment request (redirect or QR depending on payment method).

**Endpoint**: `POST /upay/transactions`

**Headers**: `Authorization` required, `x-idempotency-key` recommended

**Request Body**:

```json
{
  "emailAddress": "juandelacruz@email.com",
  "mobileNumber": "9171234567",
  "amount": 1,
  "paymentMethod": "instapay",
  "skipWhitelabelPage": true,
  "callbackUrl": "https://www.example.com/callback",
  "firstName": "Juan",
  "lastName": "Dela Cruz",
  "userRef": "USER-REF-123",
  "accountNumber": "92385921389412"
}
```

**Response** (201):

```json
{
  "code": "SP",
  "senderRefId": "TXN...",
  "uuid": "df920b19-c0fa-4ff8-a9d2-195c202e4771",
  "state": "Sent for Processing",
  "transactionId": "8834702211220159287",
  "qrCode": "0002010102122766...",
  "message": "https://.../UPAY/WhiteLabel/...",
  "paymentUrl": null,
  "status": null
}
```

Notes:

- If `paymentMethod` is `instapay` and `skipWhitelabelPage=true`, UnionBank typically returns a `qrCode`.
- For other channels, UnionBank often returns a `message` URL to redirect the payor.

### Create Debit/Credit Card Transaction

Creates a debit/credit card UPay payment request (forces UnionBank channel selection).

**Endpoint**: `POST /upay/transactions/debit-credit-card`

**Headers**: `Authorization` required, `x-idempotency-key` recommended

**Request Body**:

```json
{
  "emailAddress": "juandelacruz@email.com",
  "mobileNumber": "9171234567",
  "amount": 100,
  "skipWhitelabelPage": false,
  "callbackUrl": "https://www.example.com/callback",
  "firstName": "Juan",
  "lastName": "Dela Cruz"
}
```

**Response** (201): same shape as `POST /upay/transactions`

### Get Transaction Status

Retrieves the status of a UPay transaction.

**Endpoint**: `GET /upay/transactions/:senderRefId/status`

**Path Parameters**:

- `senderRefId`: Sender reference ID returned during creation (generated by gateway)

**Response** (200):

```json
{
  "senderRefId": "TXN...",
  "uuid": "df920b19-c0fa-4ff8-a9d2-195c202e4771",
  "status": "PENDING",
  "amount": 1,
  "paidAt": null,
  "paymentMethod": "instapay",
  "message": null
}
```

### Get Privacy Policy

Retrieves UnionBank UPay’s privacy policy content.

**Endpoint**: `GET /upay/payments/v1/privacy`

**Headers**: `Authorization` required

**Response** (200):

```json
{}
```

### Get Biller Details

Retrieves biller configuration (channels, limits, fees) by biller UUID.

**Endpoint**: `GET /upay/billers/:billerUuid`

**Response** (200):

```json
{
  "billers": [
    {
      "billerUuid": "....",
      "billerCode": "....",
      "billerName": "....",
      "accountNumber": "....",
      "paymentChannels": []
    }
  ]
}
```

### Get Biller Reference Definitions

Retrieves reference field definitions and validations configured for the biller.

**Endpoint**: `GET /upay/billers/:billerUuid/references`

**Response** (200):

```json
{
  "references": [
    {
      "billerReferenceId": "....",
      "index": "1",
      "name": "Reference No.",
      "fieldType": "ALPHANUMERIC",
      "validations": {
        "minCharLength": "1",
        "maxCharLength": "50",
        "isMasked": false,
        "isRequired": true,
        "isVisible": true,
        "fieldValidationType": { "name": "..." }
      }
    }
  ]
}
```

### Get Biller UUID Status (transactions with biller post status)

Retrieves transaction status list for a biller by UUID, including biller post status per transaction. Maps to UnionBank’s **Check Status** API (`GET /upay/payments/v1/transactions/{billerUuid}/status`).

**Endpoint**: `GET /upay/payments/v1/transactions/:billerUuid/status`

**Path Parameters**:

- `billerUuid`: Biller UUID (provided by UnionBank)

**Response** (200):

```json
{
  "code": "TS",
  "state": "Sent for Processing",
  "uuid": "a02d79d8-57a5-4130-a4aa-43258ba06cdd",
  "data": [
    {
      "status": "DEBIT_FAILED",
      "billerPostStatus": "FAILED",
      "amount": "500",
      "transactionDateTime": "2022-04-18T05:56:39Z"
    }
  ]
}
```

Notes:

- Same path is used for both **UAT** and **Sandbox**; environment is selected via `UNIONBANK_ENV` and base URL.
- Integration DTOs: `UpayBillerUuidStatusDataItem`, `UpayBillerUuidStatusResponse` in `src/integrations/unionbank/dto/response/upay-biller.response.dto.ts`.
- REST DTOs: `UpayBillerUuidStatusDataItemDto`, `UpayBillerUuidStatusResponseDto` in `src/modules/upay/dto/upay.dto.ts`.
- Endpoint constant: `UPAY_BILLER_UUID_STATUS` in `src/common/constants/unionbank-endpoints.constant.ts`.

### Get InstaPay Banks

Retrieves the list of **InstaPay receiving banks** supported by UnionBank UPay.

**Endpoint**: `GET /upay/payments/v1/instapay/banks`

**Response** (200):

```json
{
  "records": [
    {
      "code": "161207",
      "bank": "Sterling Bank Of Asia",
      "brstn": "011190019"
    }
  ]
}
```

Notes:

- The gateway returns `records` as an array (defaults to `[]` if UnionBank returns null/undefined).

### Get PESONet Banks

Retrieves the list of **PESONet receiving banks** supported by UnionBank UPay.

**Endpoint**: `GET /upay/payments/v1/pesonet/banks`

**Response** (200):

```json
{
  "records": [
    {
      "code": "161203",
      "bank": "Bank Of China",
      "brstn": "011140014"
    }
  ]
}
```

Notes:

- UnionBank’s compiled spec has an inconsistency (`record` vs `records`). The gateway DTO accepts **either** field.

## Validation Notes

Request validation is enforced via Nest global validation pipe:

- unknown fields are rejected (`forbidNonWhitelisted`)
- DTOs are transformed (`transform`)

Key validations:

- `emailAddress` must be a valid email
- `mobileNumber` must be 10–13 digits (string)
- `amount` must be \(>= 0.01\)
- `paymentMethod` (if present) must be one of:
  - `debit/credit`, `ub online`, `instapay`, `instapay p2b`, `paygate`, `gcash`, `grabpay`, `bayad_center`, `cebl`, `ecpay`, `plwn`, `mlh`, `smr`, `rds`

## Registration / Wiring

This feature is enabled by:
